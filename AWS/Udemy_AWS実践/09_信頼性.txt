Elastic IP を設定して、インスタンスが落ちた時に別のインスタンスを指すように (フローティング) する。

ELB はマネージドであり、何も設定しなくてもスケーリングする (単一障害点にならない)。

ELB の種類
- CLB (Classic Load Balancer)
	- 異なる機能のインスタンスに (機能に応じた) 振り分けができない。
	- L4/L7レイヤー (トランスポート層、アプリケーション層)
- ALB (Application Load Balancer)
	- URL のパスに基づいたルーティング (パスベースルーティング) が可能。
	- 複数ポートを個別のターゲットとして登録可能。
	- ターゲットグループでのヘルスチェックが可能。
	- L7レイヤー (アプリケーション層)
- NLB (Network Load Balancer)
	- 超低遅延の新しいやつ。
	- 毎秒数百万のリクエストに対応。
	- Pre-warming 申請が必要ない。
	- L4レイヤー (トランスポート層)

Auto-scaling のタイプ
- 垂直スケーリング (スケールアップ/ダウン)
	- メモリや CPU の強化/弱化
- 水平スケーリング (スケールアウト/イン) ←こっち
	- サーバの増加/減少

RDS のキャッシュ
- ElastiCache (インメモリ DB)
- MySQL のオプション (Memcached)
