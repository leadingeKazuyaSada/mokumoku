AWS Config
	- リソースの変更履歴を管理
	- 定期的に「構成情報」のスナップショットをとる
	- システム構成が正しいかのチェック
		- 評価基準 (AWSルール/独自ルール)
	- Stream, History, Snapshot

AWS Service Catalog
	- 承認されたサービス (CloudFormation のテンプレ) のカタログ
	- 運用管理者が作ったカタログで、権限のないユーザが環境を起動できる

AWS Artifact
	- コンプライアンス関連情報の一元管理
	- AWS Artifact Reports
		- 監査機関の基準を満たしているか、レポート作成
	- AWS Artifact Agreements
		- AWS アカウントとの契約の確認など

AWS GaurdDuty
	- ログから悪意のあるアクセスを分析
	- 機械学習による解析、レポート

AWS Systems Manager
	- 問題検出の時間短縮
		- リソースグループごとに問題をすばやく特定
	- 運用の自動化
		- パッチ、更新、設定変更、削除、停止、デプロイ
	- 可視化と制御
		- リソースグループごとの最新状態を可視化
	- ハイブリッド管理
		- AWS とオンプレを1つのインターフェイスで管理

CloudWatch
	- リソースとアプリケーションのモニタリング
	- 以下には対応しない
		- ユーザアクティビティ
		- 構成状況
	- トラブルシューティング、ログ解析
		- →自動アクション
	- 監視、ログ取得、イベントトリガによるアクション
	- EC2 などのサービスに CloudWatch エージェントを導入
		- →より詳細なログを取得できる
	- 標準メトリクスとカスタムメトリクス
		- 標準は無料、一般的なメトリクス、5分間隔
		- カスタムは有料、より多くのメトリクス、1～60秒間隔
	- CloudWatch ダッシュボード
		- RDS のメトリクスの可視化
		- RDS のダッシュボードとほぼ同じ
			- ただしカスタマイズできる
	- 拡張モニタリング
		- 50種類以上のメトリクス
	- CloudWatch アラーム
		- 特定の閾値に応じたアラーム
			- メトリクス
			- 名前空間
			- ディメンジョン
				- メトリクスを一意に識別する name-value ペア
		- SNS を通じてメール等で通知
	- CloudWatch Events
	- CloudWatch logs
		- EC2 インスタンスの場合はエージェント経由でログを転送
		- S3 にログをエクスポート可能
		- ロググループ/ログストリーム/ログイベントの3階層
		- CloudWatch logs Insights
			- 可視化・分析
			- @message, @timestamp, @logStream
	- CloudWatch ServiceLens
		- マイクロサービス単位のモニタリング
	- Contributer Insights
		- パフォーマンスに影響を与えている要因を表示
		- ルールを設定して要因を特定する
