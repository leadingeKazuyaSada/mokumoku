トランザクションの ACID
- Atomicity
	- トランザクションが「すべて実行される」か「一つも実行されない」のどちらかになる。
- Consistency
	- トランザクションの前後でデータの整合性が保たれる。
- Isolation
	- トランザクション実行中の処理が他の処理に影響しない。
- Durability
	- トランザクションが完了したら、その状態が失われない。

最適なデータベース
- リレーショナル
- NoSQL
	- キーバリューストア
		- シンプルで高速
		- 弱い整合性
	- ワイドカラムストア (列指向)
	- ドキュメントデータベース
		- mongoDB
	- グラフデータベース

DynamoDB
	- プロビジョンドスループット
		- あらかじめスループットを割り当てる
	- ストレージの容量制限がない
	- 簡易なクエリやオーダー
	- 高速、多数の同時アクセスに強い
	- JOIN/TRANSACTION/COMMIT/ROLLBACK: できない
		- 2018年11月からトランザクションに対応
	- 大量の読み書きにコストがかかる
	- 少なくとも2つの AZ で書き込み完了したら完了扱い
	- 結果整合性モデル
	- GetItem/Query/Scan: オプションで強い整合性
	- クロスリージョンレプリケーション
		- DynamoDB Streams を有効にする必要がある
	- DAX
		- インメモリ型のキャッシュ
		- 1秒間に数百万件のリクエスト処理
		- マルチ AZ

Aurora
	- 分散型の RDS (大量の読み書きに適している)
	- NoSQL のような分散処理
	- MySQL の2.5～5倍の性能
	- 商用データベースの1/10の価格
	- MySQL 5.x 互換、PostgreSQL 互換を選択可能
	- フルマネージド
	- 99.99%の高可用性・高耐久性
	- 3AZ に最大6コピーを持つが、1個の仮想ボリューム
	- 最大15個のリードレプリカ
		- マスタもマルチ可能 (Write のスケーリング)
	- MySQL からスナップショットを介してマイグレーション可能
	- Aurora サーバレス (オンデマンド自動スケーリング)
	- Aurora グローバル DB (他リージョンへの低レイテンシレプリケーション)
	
EFS (Elastic File System)
	- NAS のようなファイルストレージ
	- フルマネージド
	- NFSv4
	- ペタバイトまでスケーラブル
	- 自動でマルチ AZ
	- マウントターゲットを介して複数の EC2 から接続
		- マウントターゲットは各 EC2 側の AZ にある
		- アクセス制御はセキュリティグループでやる
	- アクセスではなくデータ量で課金
	- 汎用モードと最大I/Oモードがある
		- 同時アクセスが多い場合は最大I/Oモード
	- 高速コンピューティングが必要な場合は Amazon FSx for Lustre
	- 30日間アクセスされなかったファイルを EFS IA ストレージクラスに自動的に移行

Kinesis
	- Amazon Kinesis Data Streams
		- ストリームデータを処理するアプリを構築
	- Amazon Kinesis Data Firehose
		- ストリームデータを配信
	- Amazon Kinesis Data Analytics
		- ストリームデータを SQL クエリで分析

RedShift
	- フルマネージド型の DWH/データレイク分析サービス
	- PostgreSQL 互換の列指向データモデル
	- クラスター構成
	- 単一 AZ
	- RA3 というパフォーマンスの高いインスタンスがある
		- そうでないものを DC2 インスタンスという
	- AQUA という分散キャッシュを使える
	- 永続データとして S3 が (勝手に) 構成される
	- マテリアライズドビュー
		- SQL のビューに近い機能
	- 機械学習によりクエリの効率化を自動で行う
	- ノードを増やせる (32個まで増やせる)
	- クラスターを追加できる (10個まで追加できる)
	- RedShift Spectrum
		- ユーザ管理の S3 バケットに対してクエリエンジンが解析をする
	- クロスリージョンスナップショットすることで即座にフェイルオーバーできる。
	- 拡張 VPC ルーティングを設定して、VPC ルートテーブルで制御できる。

AWS Glue
	- ETL (Extract/Transform/Load) サービス

QuickSight
	- BI ツール

AWS Lake Formation
	- 複雑なデータレイクの構成を素早くするサービス

EMR (Elastic MapReduce)
	- マネージド型 Hadoop フレームワーク
	- 大量データを処理・分析する
