VPC を作ると必ず (最低1個) サブネットが作られる

VPC は /16～/28 の CIDR 範囲を使用できる

VPC あたりのサブネット数の上限は 200

VPC Flow Logs
	- VPC のトラフィックを取得
	- CloudWatch でモニタリング
	- 無料なので VPC を作ったら設定する

Egress Only インターネットゲートウェイ
	- IPv6 専用のインターネットゲートウェイ

Wavelength ゾーン
	- 新しくできた AZ のようなもの
	- 5G 専用のゾーン
	- キャリアゲートウェイ経由でやりとりする

インターネットからの通信 (ログイン)
	- インターネットゲートウェイ > 踏み台サーバー > プライベートサブネット
インターネットへの通信 (ダウンロードなど)
	- インターネットゲートウェイ < NATゲートウェイ < プライベートサブネット

デフォルトの ACL はデフォルトで全許可
カスタムの ACL はデフォルトで全拒否

Direct Connect
	- Direct Connect デバイスというものに自社機器を接続する
	- Direct Connect デバイスは AZ に接続されている
	- Direct Connect Gateway というリージョンを跨ぐ接続もある

VPC エンドポイント
	- グローバルIP を使って VPC 外のサービスに繋ぐための出口
	- ゲートウェイ型
		- ルートテーブルを介して接続
		- アクセス制御: エンドポイントポリシー (インライン)
	- Private Link 型
		- エンドポイント用の IP アドレスが付与される
		- アクセス制御: セキュリティグループ

VPC Peering
	- 2つの VPC 間のトラフィックルーティング
	- AWS アカウント, リージョンを跨ぐことが可能

Transit Gateway
	- 複数の VPC 間の複雑なルーティング
	- VPC を指定して Transit Gateway 接続の設定をすると、ルートテーブルが作成される	- Transit Gateway をハブとするハブアンドスポーク型
